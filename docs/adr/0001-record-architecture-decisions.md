# 0001 — Record architecture decisions

- Status: Accepted
- Date: 2026-01-01

## Context

We want the *reasons* behind significant choices to survive turnover and time,
so future contributors can change things deliberately rather than fearfully.

## Decision

Use Architecture Decision Records (ADRs): one short Markdown file per
significant decision, numbered and immutable. Supersede rather than edit.

## Consequences

- A lightweight, greppable history of "why".
- Minimal overhead — a few paragraphs per decision.
